# Data model

Master data covers items and revisions, BOMs and lines, routings and operations, suppliers, approved vendors, supplier items, and packaging. Operational exposure covers POs, work orders, inventory, and customer orders. Governance data covers packets, documents, extraction results, findings, duplicate candidates, impacts, proposed change sets, approvals, posting transactions, escalations, and immutable audit events.

Foreign keys, unique record identifiers, case versions, timestamps, explicit statuses, and staged/posting separation make workflow invariants inspectable. JSON is limited to evidence, snapshots, typed candidate values, and graph paths.
